发表于 2017-04-13 | 分类于 我有一个geek梦 | | 阅读次数 |

1 名词解释

1.1 节点（node）
节点都是各自独立的可执行文件。它主要执行计算处理，能够通过话题，服务或参数服务器与其他进程（节点）通信。  
ROS被设计为细粒度的模块化的系统：一个机器人控制系统通常有很多节点主持人组成。  
 ROS通过使用节点将代码和功能解耦，提高了系统容错能力和可维护性，使系统简化。
1.2 话题（topic）
话题是节点间数据传输的总线。通过主题进行消息交流，不需要直接交流。（彼此无直接联系）。  
某个节点发送key strokes到一个话题上，与此同时，另外一个节点可以通过该话题获得key strokes。
值得注意的是每个主题都是强类型的。  
    1.2.1发布到主题上的消息必须与主题的ROS消息类型相匹配，并且节点只能接收类型匹配的消息。
    1.2.2节点通信，必须具有相同的消息类型。
